# ТЕХНИЧЕСКОЕ ЗАДАНИЕ (MVP)

### 1. Цель
Разработать интерактивную веб-платформу с интегрированной WebGL-игрой, реконструирующей исторический эпизод в машинно-котельном отделении корабля. Главная цель проекта — проверка образовательной гипотезы: прохождение интерактивного эпизода от первого лица повышает первичный интерес пользователя к реальным историческим событиям.

**Ключевые задачи ТЗ:**
* Реализовать веб-сайт, служащий входной точкой (лендинг) и контейнером для WebGL-игры.
* Разработать игровую механику от первого лица (Unity), включающую исследовательскую часть, систему задач и кризисную ситуацию (пожар, повышение температуры).
* Обеспечить нелинейность прохождения: выбор игрока в кризисной ситуации должен приводить к различным концовкам.
* Интегрировать блок исторической справки (включая текстовые материалы и короткий видеоролик) на финальном экране.

**Форма поставки (MVP):**
Рабочий веб-сайт с интегрированным WebGL-билдом игры, развёрнутый на сервере, доступный через десктопные браузеры без необходимости установки локальных файлов.

---

### 2. Описание информационной системы
Проект имеет клиент-серверную архитектуру (с акцентом на клиентскую часть) и состоит из веб-оболочки и встроенного игрового движка. Поскольку бóльшая часть функционала уже реализована командой, текущая архитектура отражает актуальное состояние сборки.

| Компонент | Описание |
| :--- | :--- |
| **Сайт проекта (Фронтенд)** | Веб-интерфейс (HTML/CSS/JS), содержащий главную страницу, контейнер для WebGL, блок исторической справки (текст + видео) и скрипты сбора аналитики. |
| **Игровой клиент (Unity)** | Скомпилированный WebGL-билд игры, загружаемый в браузер пользователя. |

**Игровые модули (Unity WebGL):**

| Модуль | Назначение |
| :--- | :--- |
| **GameManager** | Управление общим состоянием игры, связывание систем. |
| **PlayerController** | Перемещение игрока и управление камерой от первого лица. |
| **InteractionSystem** | Обработка взаимодействия с объектами (вентиль, котёл, трубы и др.). |
| **ChecklistManager** | Контроль списка задач (бортовой журнал) и их выполнения. |
| **TemperatureManager** | Отслеживание температуры окружающей среды и состояния игрока. |
| **UIManager** | Отрисовка HUD, температуры, чеклиста, подсказок и эффектов состояния. |
| **AudioManager** | Воспроизведение звуков (окружение корабля, сирена, шаги, огонь, приборы). |
| **CrisisManager** | Инициация и контроль кризисной части (пожар, дым, рост температуры). |
| **EndingController** | Обработка финальных выборов игрока, вывод исхода и переход к справке. |

### 4. Нефункциональные требования

**Производительность и стабильность**

| ID | Требование |
| :--- | :--- |
| N1 | Игра должна обеспечивать стабильную частоту кадров в браузере (не менее 30 FPS в сценах с эффектами пожара и дыма). |
| N2 | Время первичной загрузки WebGL-билда должно быть оптимизировано (насколько позволяет вес ассетов). |
| N3 | Перезапуск игры без обновления страницы (или при повторном запуске) не должен вызывать критических ошибок или утечек памяти. |

**Совместимость и доступность**

| ID | Требование |
| :--- | :--- |
| N4 | Приложение должно корректно работать в десктопных версиях браузеров: Google Chrome, Firefox, Яндекс Браузер. |
| N5 | Управление должно осуществляться стандартными паттернами (WASD + мышь) и быть интуитивно понятным. |
| N6 | Пользовательский интерфейс должен быть легко читаемым, не перегруженным и не отвлекающим от погружения. |

**Историческая достоверность**

| ID | Требование |
| :--- | :--- |
| N7 | Сюжет, объекты окружения и справочные материалы (текст и видео) должны строго соответствовать собранным историческим данным, не искажая смысл реального подвига. |

### 5. Критерии приёмки

**Геймплей и механики**
* **AC1:** Игрок может загрузить игру на сайте и пройти этап спокойного исследования, взаимодействуя с котлом, вентилем и цистерной.
* **AC2:** Бортовой журнал корректно обновляется при выполнении задач.
* **AC3:** Кризисное событие (пожар) запускается без сбоев, сопровождается падением видимости (дым), визуальными (огонь) и звуковыми (сирена) эффектами.
* **AC4:** Температурная шкала реагирует на окружающую среду.
* **AC5:** Игра успешно считывает порядок действий игрока и выдает соответствующий экран концовки.

**Сайт и аналитика**
* **AC6:** После показа игровой концовки корректно отображается предложение перейти к исторической справке.
* **AC7:** Историческая справка содержит текстовую информацию и встроенный короткий видеоролик, которые воспроизводятся без ошибок.
* **AC8:** Клик по справке успешно регистрируется в базе для последующего расчета метрики гипотезы.

### 6. DEFINITION OF DONE (Критерии готовности)

* [ ] Актуальный билд Unity скомпилирован под WebGL и интегрирован в финальную версию сайта.
* [ ] Все заглушки (устаревший контекст) в коде и текстах заменены на финальные исторические и игровые данные.
* [ ] Проведено тестирование производительности: игра выдает стабильные 30+ FPS в сценах с частицами (огонь, дым) в Chrome, Firefox и Яндекс Браузере.
* [ ] Система аналитики подключена и корректно фиксирует переходы к экрану справки.
* [ ] Проведено фокус-тестирование на целевой аудитории для проверки гипотезы проекта.
* [ ] Собрана статистика: гипотеза признается подтвержденной, если **>60%** пользователей, прошедших игру до конца, добровольно открыли историческую справку.
* [ ] Проект (веб-сайт + игра) развернут на публичном/тестовом сервере и готов к релизу.
